# Task 21 - Azure Entra ID (Users, Roles, Basic Identity Management)

## 📌 Objective
To manage identities and assign roles using Azure Entra ID (formerly Azure Active Directory).

This task demonstrates basic Identity and Access Management (IAM) concepts in Azure.

---

## 🛠️ Services Used
- Microsoft Azure Portal
- Azure Entra ID
- Role-Based Access Control (RBAC)

---

## 👤 Implementation Steps

### Step 1: Create New User

1. Logged into Azure Portal.
2. Navigated to **Azure Entra ID**.
3. Clicked **Users → New User**.
4. Selected **Create User**.
5. Entered:
   - Name
   - Username
   - Password
6. Clicked **Create**.

User was successfully created in Entra ID.

---

### Step 2: Assign Role to User

1. Opened the created user.
2. Navigated to **Assigned Roles**.
3. Clicked **+ Add Assignment**.
4. Selected a role (e.g., Reader or Contributor).
5. Clicked **Assign**.

Role assignment completed successfully.

---

## 🔐 Example Role Types

- **Global Administrator** – Full control over tenant.
- **Contributor** – Can manage resources but not assign roles.
- **Reader** – View-only access.

## 🔍 Key Learning

- Azure Entra ID manages identities in Azure.
- Users can be created and managed centrally.
- RBAC controls what actions a user can perform.
- IAM is critical for security and governance.
- Similar concept to AWS IAM.
