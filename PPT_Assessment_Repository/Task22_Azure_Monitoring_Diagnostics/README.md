# Task 22 - Azure Monitoring & Diagnostics (Basic)

## 📌 Objective
To enable monitoring and diagnostics for Azure resources using Azure Monitor and analyze performance metrics.

This task demonstrates how cloud resources are monitored for performance, availability, and troubleshooting.

---

## 🛠️ Services Used
- Microsoft Azure Portal
- Azure Monitor
- Metrics & Diagnostic Settings
- Log Analytics (optional)

---

## 📊 Implementation Steps

### Step 1: Enable Monitoring

1. Logged into Azure Portal.
2. Selected an existing resource (e.g., Virtual Machine).
3. Opened **Monitoring → Metrics**.
4. Viewed real-time performance data such as:
   - CPU Percentage
   - Disk Read/Write Operations
   - Network In/Out

Metrics were successfully displayed.

---

### Step 2: Configure Diagnostic Settings

1. Opened the selected resource.
2. Navigated to **Diagnostic Settings**.
3. Clicked **Add Diagnostic Setting**.
4. Selected:
   - Send logs to Log Analytics Workspace (optional)
   - Enable performance logs
5. Saved configuration.

Diagnostics enabled successfully.

## 🔍 Key Learning

- Azure Monitor tracks performance and health of resources.
- Metrics provide real-time performance data.
- Diagnostic settings collect logs for troubleshooting.
- Helps in performance optimization and issue detection.
- Essential for production environments.
