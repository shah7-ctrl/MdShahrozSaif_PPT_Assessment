## 📊 Part 1: Service Level Agreement (SLA)

### What is SLA?

Service Level Agreement (SLA) defines the guaranteed availability percentage of an Azure service.

Example:
- 99.9% availability → Allows limited downtime per month.
- Higher SLA → Lower allowed downtime.

### Steps Performed:

1. Logged into Azure Portal.
2. Navigated to any Azure service (e.g., Virtual Machine).
3. Opened official SLA documentation from Microsoft.
4. Reviewed availability percentage and uptime commitment.

---

## 🔄 Part 2: Service Lifecycle

Azure services follow different lifecycle stages:

- Preview → Under testing, limited support.
- General Availability (GA) → Fully supported, production-ready.
- Deprecated → Being phased out.

### Steps Performed:

1. Explored Azure services.
2. Checked lifecycle status (Preview / GA).
3. Understood production readiness considerations.

---

## 💳 Part 3: Subscription Management

### What is Subscription?

A subscription in Azure is a billing and access boundary that:

- Tracks usage and cost.
- Controls resource access.
- Applies policies and budgets.

### Steps Performed:

1. Opened **Subscriptions** in Azure Portal.
2. Viewed:
   - Subscription ID
   - Billing details
   - Resource usage
3. Explored role assignments and access control.
